Mantener separado el código el controlador de la vista facilita la estructuración de los programas, permite un mayor nivel de reutilización de código e independentiza el proceso de desarrollo de ambas partes.

$_SERVER contiene información sobre cabeceras, rutas y ubicaciones de scripts proporcionada por el servidor, $_SESSION lleva las variables de sesión que están disponibles para el script que se está ejecutando. $_FILES contiene elementos subidos al script con el método POST. $_ENV es un array con información sobre el ambiente en que se ejecuta el script.

count() cuenta los elementos de cualquier array (o diccionario) de php. checkdate() revisa si una fecha en formato mdy es válida.

Un ataque de XSS (cross-site scripting) involucra la introducción de un script malicioso por parte de un usuario que afecta al sitio al momento de mostrarse. Cualquier entrada de datos efectuada por un usuario debe ser validada antes del guardado y convertida a entidades html antes de mostrarla en la página. Si los datos contienen un script y este no se convierte, las funciones como "echo" que suelen utilizarse para mostrar html al cliente lo interpretarán como un programa incrustado en el documento y afectarán el sitio. Estos ataques pueden prevenirse convirtiendo los caracteres de las etiquetas en entidades html a través de las funciones htmlentities() (que busca y convierte en entidades todos los caracteres que pudiesen causar problemas al sitio) y htmlspecialchars() (que solo convierte los caracteres más problemáticos).

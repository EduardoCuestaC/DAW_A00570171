El sexto botón corre en la consola una demostración de herencia y creación de objetos con prototipos en javascript.


Semejanzas y diferencias entre Java y Javascript:

La orientación a objetos en Javascript no utiliza el mismo sistema de clases que Java, sino que hace uso de prototipos para establecer funciones miembro, así como crear herencias y polimorfismo.

Métodos para objetos Date:

  • .toString(): regresa un string con el día de la semana, mes, día del mes, año, hora y zona horaria.
  • .toUTCString(): regresa un string en el formato UTC ("día, núm.día mes año horas:minutos:segundos zona horaria.
  • .now(): regresa el número de milisegundos que han pasado desde el primero de enero de 1970.
  • .setHours(): permite ajustar la hora del objeto Date sin alterar minutos ni segundos.
  • .setYear(): ajusta el año del objeto sin modificar los demás valores.

Métodos para objetos Array

  • .filter(): recibe una función como parámetro y regresa un array con los elementos que hicieron que la función regresara true.
  • .map(): recibe una función como parámetro y construye un nuevo array con los valores que regresan de ella.
  • .forEach(): recibe una función como parámetro y la ejecuta con cada uno de los elementos del array.
  • .slice(): regresa una copia del array original que solo contiene los elementos desde el inicio hasta el índice especificado como parámetro. No modifica al array desde el que es llamada (al contrario de .splice()).
  • .reverse(): invierte el orden de los elementos del array.

Variables de enfoque local en javascript

Usando let en lugar de var puede declararse una variable o función que únicamente existe dentro del bloque de código en que fue creada. Es recomendable utilzarla únicamente para variables temporales dentro de ciclos de control para evitar conflictos en otras partes del código si se utilizan los mismos nombres.

Declarar una variable como var dentro de una función le da un enfoque local, por lo que no afecta al resto del código

Expresiones regulares en strings

Puede usarse el método -string-.match(-expresión regular-) para obtener la primera sección de un string que contiene un formato específico (descrito por el objeto de expresión regular). También puede usarse -expresión regular-.test(-string-) para comprobar si un string cumple con cierto formato. Las expresiones regulares son útiles en aplicaciones web para comprobar el formato de los campos de una forma. Es posible escribir expresiones regulares que describan teléfonos, direcciones, correos electrónicos, etc. y luego probarlas contra los valores que ingresa el usuario para asegurar que la información obtenida por la forma satisfaga las condiciones que requiere la aplicación para procesarla adecuadamente.
